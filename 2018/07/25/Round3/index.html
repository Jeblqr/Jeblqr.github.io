#OneCode天梯赛 Round3

第一题
困难的编码

原题链接：http://oj.onecode.com.cn/index.php/Ladder/showProblemPage/id/10132

时间限制: 1000 MS 内存限制: 131072 KB

问题描述

OneCode收到了一份奇怪的编码，据说这是先知的一份笔记，这个笔记让人难以理解，不过聪明的CXK很快就发现了这个编码的规律。
这个规律是，这个编码的总长度是两个数的乘积，并且把这个编码放在一个N*M的格子里，就可以看出其中的关联，其中，N和M是两个整数，M是较大的那个数。
现在CXK想要立刻知道这个编码所代表的含义，但是它现在身边没有电脑，你能帮他解决这个问题吗？

输入格式

第一行是两个整数n和m(1<=n,m<=1000)。

接下来一行有一个字符串s，保证长度为n*m,且只由字母组成。

其中数据范围:(1<=n,m<=1000000),(0<=k<=1000000)(1<=x<=n,1<=y<=m)

输出格式

输出n*m的表示法。

样例输入

4 5

ChuTiRenShiZhiZhangA

样例输出

ChuTi
RenSh
iZhiZ
hangA

用string+substr方法，时间复杂度不高，直接AC

第二题
贪吃的ZZ恺成

原题链接：http://oj.onecode.com.cn/index.php/Ladder/showProblemPage/id/10133

时间限制: 1000 MS 内存限制: 131072 KB

问题描述

ZZ恺成是个大胃王，有一天，算法“状态规划”发明者，高精度加法不会写者，张焰，买了一大堆零食。她把零食分成了n堆，其中第i堆有i包零食。
也就是说，第1堆有1包零食，第2堆有两包零食，第3堆有3包零食……第n堆有n包零食。然后，由于恺成比较ZZ，每次只会从1堆或者多堆里取走同样多包的零食吃掉。
那么，ZZ恺成至少需要多少次就可以把n堆零食拿完呢？
比如：n=3。那么，一开始零食的状态是1,2,3，先把2,3堆取走2，然后状态变成1,0,1，接着，只要取走1,3堆的1包即可完成。

输入格式

第一行是一个整数t(1<=t<=10^5),代表接下来有t组测试数据。

对于每一组测试数据，只包含一个整数n(1<=n<=10^18)。

输出格式

对于每组测试数据，输出结果，并且占一行。

样例输入

5
1
2
3
4
5

样例输出

1
2
2
3
3

很难看出，这道题有一个不易得的规律：

如果n为奇数，ans(n)=ans((1+n)/2-1）+1;
如果n为偶数，ans(n)=ans((2+n)/2-1）+1;

？？？

咳咳

ans(n)=ans(n/2)+1;

边界条件为:

ans(1)=1;
ans(2)=2;

考虑到该题数据不掺水，采用一丢丢二进制优化以及递归实现，时间复杂度为

O(t*lgn)

第三题
小强的球

原题链接：http://oj.onecode.com.cn/index.php/Ladder/showProblemPage/id/10134

时间限制: 1000 MS 内存限制: 131072 KB

问题描述

小强很喜欢玩球，他想要在桌子上摆一些球，排成一排。
球只有三种颜色，红色，黄色，或者蓝色。现在，小强有R个红球，Y个黄球，B个蓝球，需要依次摆在桌子上。
小强每放一个新的球，可以选择放在球的中间或者放在端头。
每放一个球，小球就可以得到一个分数。计分规则如下：
1.对于刚开始放的第一个球，计0分。
2.如果球放在端头，则分数等于之前放过的球的颜色数，例如之前放球的状态是RYYR，现在把球放在末尾，则计2分。
3.如果球放在某两个球之间，则分数为前半段的球的颜色加上后半段的球的颜色数，例如之前放球的状态是RYYR，现在把球放在两个Y中间，则计4分，因为前半段有两种颜色，后半段也有两种颜色。

那么，小强放球能获取的最大值是多少呢？

输入格式

第一行是一个整数t(1<=t<=10^5),代表有t组测试数据。

接下来t行，每行有三个整数R,Y,B(0<=R,Y,B<=10^9)，分别代表每种颜色球的个数，含义跟题目描述中的一致。

输出格式

对于每组测试数据，输出结果，并且占一行。

样例输入

3
2 2 2
3 3 3
4 4 4

样例输出

15
33
51

不会，提交了样例，略过。

听了同学讲解，此题需要数学归纳，再写分段函数（或者说贪心+特例），用O(1)的复杂度快速刷爆

```cpp

#include

#include
using namespace std;
unsigned long long calc(unsigned long long a,unsigned long long b,unsigned long long c)
{
if (a==1&&b==1&&c==1)//边界
return 3;
if ((a==1||a==0)&&b==0&&c==0)//边界
return 0;
if (a==1&&b==1&&c==0)//边界
return 1;
if (a>=2&&b==0&&c==0)//左/右1,另一边1，共2分
return 1+2(a-2);
if (a>=2&&b==1&&c==0)//左/右2,另一边1，共3分
return 3+3(a-2);
if (a>=2&&b==1&&c==1)//左/右3,另一边1，共4分
return 10+4(a-2);
if (a>=2&&b>=2&&c==0)//左右二色，共4分
return 6+4(a+b-4);
if (a>=2&&b>=2&&c==1) //左/右3,另一边2，共5分
return 10+5(a+b-4);
return 15+6(a+b+c-6);//左右三色，共6分
}
int main()
{
int n;
cin>>n;
unsigned long long a[5];
for (int i=1;i<=n;i++)
{
cin>>a[1]>>a[2]>>a[3];
sort(a+1,a+4);
cout<<calc(a[3],a[2],a[1])<<endl;
}
return 0;
}

涯涯最近在研究算法，他遇到了一个困难的问题，有n个由小写字母组成的字符串。现在要选择尽可能多的字符串，使得每个字母都出现偶数次。
比如有6个字符串 abd,eg,ge,abe,ac,bcd。编号为1,2,3,4,5,6.显然选择第1,2,3,5,6个。

输入格式

第一行是一个数字n(1<=n<=24)。代表接下来有n行字符串。

接下来n行每行有一个只由小写字母组成的字符串。每个字符串的长度不超过10^5。

输出格式

输出最多能选字符串的个数，并按从小到大的顺序输出每个字符串的编号，假设字符串编号从1开始。
